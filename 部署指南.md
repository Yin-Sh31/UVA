# NoPerson 项目部署指南

## 环境准备

### 必需软件

| 软件 | 版本要求 | 用途 |
|------|---------|------|
| JDK | 17+ | 后端运行 |
| Maven | 3.6+ | Java 构建工具 |
| Node.js | 16+ | 前端开发 |
| npm | 8+ | 前端依赖管理 |
| MySQL | 8.0+ | 关系型数据库 |
| Redis | 6.0+ | 缓存服务 |
| 微信开发者工具 | 最新版 | 小程序开发 (可选) |

### 推荐配置

- **操作系统**: Linux (Ubuntu 20.04+) / macOS / Windows 10+
- **内存**: 8GB 以上 (建议 16GB)
- **磁盘**: 20GB 可用空间

## 第一步：数据库初始化

### 1.1 安装 MySQL

**Ubuntu/Debian:**
```bash
sudo apt update
sudo apt install mysql-server-8.0
sudo systemctl start mysql
sudo systemctl enable mysql
```

**macOS (Homebrew):**
```bash
brew install mysql@8.0
brew services start mysql@8.0
```

**Windows:**
从 [MySQL 官网](https://dev.mysql.com/downloads/mysql/) 下载安装包进行安装。

### 1.2 创建数据库和用户

```sql
-- 登录 MySQL
mysql -u root -p

-- 创建数据库
CREATE DATABASE noperson CHARACTER SET utf8mb4 COLLATE utf8mb4_0900_ai_ci;

-- 创建用户并授权 (生产环境请修改密码)
CREATE USER 'noperson'@'localhost' IDENTIFIED BY 'your_password_here';
GRANT ALL PRIVILEGES ON noperson.* TO 'noperson'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

### 1.3 导入数据表结构

```bash
# 进入项目目录
cd noperson-finally

# 导入主 SQL 文件
mysql -u noperson -p noperson < db_migration/noperson.sql

# 依次导入增量更新脚本 (按文件名顺序)
mysql -u noperson -p noperson < db_migration/create_banner_table.sql
mysql -u noperson -p noperson < db_migration/create_device_activity_table.sql
mysql -u noperson -p noperson < db_migration/create_device_rental_table.sql
mysql -u noperson -p noperson < db_migration/create_farmer_favorite_address.sql
mysql -u noperson -p noperson < db_migration/create_feedback_table.sql
mysql -u noperson -p noperson < db_migration/create_message_table.sql
mysql -u noperson -p noperson < db_migration/create_payment_tables.sql
mysql -u noperson -p noperson < db_migration/create_system_config_table.sql
mysql -u noperson -p noperson < db_migration/create_transaction_record.sql
mysql -u noperson -p noperson < db_migration/add_flyer_complaint.sql
mysql -u noperson -p noperson < db_migration/add_flyer_evaluation.sql
mysql -u noperson -p noperson < db_migration/add_payment_fields.sql
mysql -u noperson -p noperson < db_migration/add_rental_status.sql
mysql -u noperson -p noperson < db_migration/alter_feedback_table.sql
mysql -u noperson -p noperson < db_migration/create_admin_table.sql
mysql -u noperson -p noperson < db_migration/update_amount_decimal.sql

# (可选) 导入测试数据
mysql -u noperson -p noperson < db_migration/test_data.sql
mysql -u noperson -p noperson < db_migration/test_demand_data.sql
```

### 1.4 验证数据库

```sql
mysql -u noperson -p noperson -e "SHOW TABLES;"
```

应看到 30+ 张表。

## 第三步：后端服务部署

### 3.1 配置文件创建

在 `noperson-finally/src/main/resources/` 目录下创建 `application.yml`:

```yaml
server:
  port: 8082

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/noperson?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: noperson
    password: your_password_here
    driver-class-name: com.mysql.cj.jdbc.Driver
  
  redis:
    host: localhost
    port: 6379
    password:  # 如有密码请填写
    database: 0
    timeout: 10000ms

  servlet:
    multipart:
      max-file-size: 50MB
      max-request-size: 50MB

# JWT 配置
jwt:
  secret: your_jwt_secret_key_here_please_change_in_production
  expiration: 86400000  # 24 小时

# 阿里云 OSS 配置 (如使用本地存储可暂不配置)
aliyun:
  oss:
    endpoint: oss-cn-hangzhou.aliyuncs.com
    access-key-id: your_access_key_id
    access-key-secret: your_access_key_secret
    bucket-name: your_bucket_name

# 第三方 API 配置
map:
  amap:
    key: your_amap_key  # 高德地图 API Key
  tencent:
    key: your_tencent_key  # 腾讯地图 API Key

weather:
  qweather:
    key: your_qweather_key  # 和风天气 API Key

# 文件上传路径
upload:
  path: ./upload

# MyBatis-Plus 配置
mybatis-plus:
  configuration:
    map-underscore-to-camel-case: true
    log-impl: org.apache.ibatis.logging.stdout.StdOutImpl
  global-config:
    db-config:
      id-type: auto

# Swagger/OpenAPI 配置
springdoc:
  api-docs:
    enabled: true
  swagger-ui:
    enabled: true
    path: /swagger-ui.html
```

### 3.2 编译打包

```bash
cd noperson-finally

# 清理并编译
mvn clean package -DskipTests

# 或直接运行
mvn spring-boot:run
```

### 3.3 启动后端服务

**方式一：Maven 直接运行 (开发环境)**
```bash
cd noperson-finally
mvn spring-boot:run
```

**方式二：JAR 包运行 (生产环境)**
```bash
cd noperson-finally
java -jar target/NoPerson-finally-0.0.1-SNAPSHOT.jar
```

**方式三：后台运行 (Linux)**
```bash
nohup java -jar target/NoPerson-finally-0.0.1-SNAPSHOT.jar > logs/app.log 2>&1 &
```

### 3.4 验证后端服务

访问以下地址验证:
- API 根路径: http://localhost:8082
- Swagger UI: http://localhost:8082/swagger-ui.html
- 健康检查: http://localhost:8082/auth/login (应返回登录页面或错误信息，但服务可达)

查看日志:
```bash
tail -f logs/app.log
```

## 第七步：生产环境部署

### 7.1 后端生产配置

创建 `application-prod.yml`:

```yaml
server:
  port: 8082

spring:
  datasource:
    url: jdbc:mysql://your-db-host:3306/noperson?useUnicode=true&characterEncoding=utf8&useSSL=true&serverTimezone=Asia/Shanghai
    username: ${DB_USERNAME}
    password: ${DB_PASSWORD}
  
  redis:
    host: ${REDIS_HOST}
    port: 6379
    password: ${REDIS_PASSWORD}

jwt:
  secret: ${JWT_SECRET}  # 使用环境变量
  expiration: 86400000

aliyun:
  oss:
    endpoint: ${OSS_ENDPOINT}
    access-key-id: ${OSS_ACCESS_KEY_ID}
    access-key-secret: ${OSS_ACCESS_KEY_SECRET}
    bucket-name: ${OSS_BUCKET_NAME}

upload:
  path: /var/www/noperson/upload
```

启动时指定环境:
```bash
java -jar target/NoPerson-finally-0.0.1-SNAPSHOT.jar --spring.profiles.active=prod
```

### 7.2 Nginx 反向代理配置

安装 Nginx:
```bash
sudo apt install nginx  # Ubuntu/Debian
```

配置 `/etc/nginx/sites-available/noperson`:

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # Web 前端
    location / {
        root /path/to/frontend/dist;
        try_files $uri $uri/ /index.html;
    }

    # 管理后台
    location /admin {
        alias /path/to/admin-frontend/dist;
        try_files $uri $uri/ /index.html;
    }

    # 后端 API
    location /api {
        proxy_pass http://localhost:8082;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 静态资源 (上传文件)
    location /upload {
        alias /path/to/noperson-finally/upload;
        expires 30d;
    }
}
```

启用配置:
```bash
sudo ln -s /etc/nginx/sites-available/noperson /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

### 7.3 SSL 证书配置 (HTTPS)

使用 Let's Encrypt 免费证书:
```bash
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```

## 常见问题排查

### 问题 1: 后端启动失败 - 数据库连接错误

**症状**: `Communications link failure`

**解决方案**:
1. 确认 MySQL 服务已启动: `sudo systemctl status mysql`
2. 检查数据库用户名密码是否正确
3. 确认数据库已创建并导入表结构
4. 检查防火墙是否开放 3306 端口

### 问题 2: 后端启动失败 - Redis 连接错误

**症状**: `Unable to connect to Redis`

**解决方案**:
1. 确认 Redis 服务已启动: `sudo systemctl status redis`
2. 检查 Redis 配置: `redis-cli ping`
3. 如有密码，确保配置文件中已填写

### 问题 3: 前端无法调用 API

**症状**: 浏览器控制台显示 CORS 错误或网络错误

**解决方案**:
1. 确认后端服务已启动且可访问
2. 检查 Vite 代理配置是否正确
3. 生产环境检查 Nginx 反向代理配置

### 问题 4: 401 未授权错误

**症状**: 登录后仍然提示未授权

**解决方案**:
参考 [README_FLYER_PROFILE_DEBUG.md](./noperson-finally/README_FLYER_PROFILE_DEBUG.md) 详细排查指南。

### 问题 5: 文件上传失败

**症状**: 上传图片或文件时失败

**解决方案**:
1. 检查 `upload` 目录是否存在且有写权限
2. 检查 `application.yml` 中的上传路径配置
3. 如使用 OSS，检查 AK/SK 配置是否正确
